# 原生平台行为书

本文约束 Capacitor iOS/Android 壳提供的系统级交互。业务命令、模式、视图和任务层级仍由 Web 应用及各自行为书定义，原生桥只增强输入反馈和设备集成。

## 会话与网络

- **NATIVE-001**：移动壳只连接构建时指定的单一公共 HTTPS origin；缺失、非 HTTPS 或包含路径的地址必须使移动构建失败。
- **NATIVE-002**：原生登录使用独立 Bearer token，不复用浏览器 cookie、MCP key 或浏览器 remember token。
- **NATIVE-003**：勾选记住登录时，token 只写 Android Keystore 加密存储或 iOS Keychain；未勾选时只保留在当前进程内存。
- **NATIVE-004**：原生 API 返回 401、用户退出或账号会话重置时清除设备 token。安全存储暂时无法清除时记录待清理状态，后续启动必须先重试且不得读取旧 token。修改密码撤销旧 token，并只为当前会话签发替换 token。

## 键盘、返回与安全区

- **NATIVE-101**：系统键盘出现时隐藏移动底栏、让编辑控件进入可视区域，并由系统 resize 与安全区共同决定内容空间；键盘关闭后恢复布局。
- **NATIVE-102**：Android 返回键优先级为：退出当前文本编辑并收起键盘 → 关闭顶层弹窗/临时界面 → 回到应用内上一个底部页签 → 回到列表页签 → 最小化应用。
- **NATIVE-103**：状态栏和导航栏图标跟随主题明暗；内容不得依赖固定刘海、Home indicator 或系统栏像素值。

## 触觉反馈

- **NATIVE-201**：触觉反馈默认开启，可在移动端“更多”中关闭；关闭不改变任何业务行为。
- **NATIVE-202**：触摸长按真正提起任务时仅反馈一次；只有预计落位的语义目标改变时才产生 selection 反馈，指针逐帧移动不得重复震动。
- **NATIVE-203**：有效落位、无效落位和取消分别结束一次 selection 会话；鼠标、浏览器和 Electron 不调用原生触觉反馈。
- **NATIVE-204**：任何原生反馈 API 失败都静默降级，不阻断拖拽、编辑、导航或保存。

## 发布边界

- **NATIVE-301**：Android/iOS 工程是可审查的源代码；Web 资源通过 `build:mobile` 重新构建并同步，不手工修改平台工程中的 `public` 产物。
- **NATIVE-302**：Windows + JDK 21 可生成和验证 Android 工程；iOS 最终编译、签名和真机验证必须在 macOS/Xcode 完成。
