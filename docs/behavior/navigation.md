# 模式与视图行为书

本文是工作区导航行为的唯一事实来源。模式与视图是两个独立维度。

## 状态

| 模式 | 允许的视图 | 数据来源 |
|---|---|---|
| 清单模式（`checklist`） | 仅列表视图 | 系统清单页 |
| 页面模式（`page`） | 列表视图、依赖图视图 | 用户页面 |

页面模式在会话内保存返回上下文 `{ pageId, view }`。退出登录或切换账号时清除；不写入工作区文件。

## 转换规则

- **NAV-001**：从页面模式进入清单模式前，记录当前页面和当前视图。
- **NAV-002**：从清单模式返回页面模式时，恢复 NAV-001 记录的页面和视图；页面已删除时回退到第一个用户页面，视图保持不变。
- **NAV-003**：清单模式永远显示列表视图，不能进入依赖图。
- **NAV-004**：页面模式内切换页面时保持当前视图。
- **NAV-005**：新建页面后进入该页面的列表视图。
- **NAV-006**：页面切换未完成时忽略重复切换，失败时保留原上下文。
- **NAV-007**：`更多`是移动端临时界面，不是工作区模式或视图，离开后恢复此前视图。
- **NAV-008**：清单模式不显示 `Blocked` 分组；系统清单页没有依赖关系，未完成任务统一属于 `Ready`。

`meta.activePageId` 表示当前加载的数据页，不得再被当作页面模式的返回历史；返回历史由客户端导航状态单独持有。
