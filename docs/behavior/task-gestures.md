# 任务手势行为书

## 列表视图

| 输入 | 结果 |
|---|---|
| 单击标题 | 不进入编辑；链接仍可打开 |
| 320ms 内双击标题 | 行内编辑，在第二次点击附近放置光标 |
| 长按标题或行内非交互区域 400ms | 提起任务并开始拖动，不进入编辑 |
| 长按前移动超过 8px | 取消长按；纵向交给滚动，明确横向进入滑动 |
| 向右滑动超过 96px | 尝试完成任务 |
| 点击描述按钮 | 切换描述编辑区；第二次点击关闭 |

- **GEST-001**：按钮、输入框和文本域自行处理点击，不启动行拖动。
- **GEST-002**：拖动激活后阻止列表和页面滚动；结束、取消、多指加入、失去捕获或卸载都必须恢复滚动并清空会话。
- **GEST-003**：首次拖动、连续拖动同一任务和随后拖动其他任务必须使用同一状态转换，不能保留上次 pointer/touch 状态。
- **GEST-004**：拖动浮层保留触点相对任务行的偏移，不能跳到触点左下方。
- **GEST-005**：有效落位始终显示预计插入线或嵌套框；提交后使用位置动画表现变化。
- **GEST-006**：焦点/按压高亮在失焦后清除；推荐任务的紫色语义高亮只随推荐状态变化，不显示说明文字。

桌面鼠标在移动超过 8px 后开始拖动；双击标题进入相同的行内编辑。桌面不要求长按。

## 依赖图视图

- **GEST-101**：节点拖动、空白平移和双指缩放由 React Flow 手势负责。
- **GEST-102**：空白区域单指静止长按 500ms 创建任务；移动超过 10px 或出现第二根手指即取消。
- **GEST-103**：只在可拖动区域禁用系统触摸菜单和文本选择，输入区保持原生编辑能力。
