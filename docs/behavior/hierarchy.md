# 层级与拖拽行为书

父子层级与依赖连线是两个不同关系；最大父子深度为三级。

## 父节点类型

- **HIER-001**：无直接子节点的是普通任务节点。
- **HIER-002**：有 1–10 个直接子节点的是普通父节点。
- **HIER-003**：加入第 11 个直接子节点后，在依赖图中成为折叠父节点。父节点可见，内部后代从画布隐藏，并可从父节点内容区访问。
- **HIER-004**：列表视图不因数量自动折叠；展开/收起仅是当前列表界面的手动状态。
- **HIER-005**：父任务有未完成的直接子任务时不可标记为完成；未完成任务也不可移入已完成父任务。此规则对界面、REST API 和 MCP 写入一致生效。

## 列表落位

- **DROP-001**：落在目标行上/下边缘，成为目标行同级并插入对应一侧；这条规则允许三级节点提升到二级同级。
- **DROP-002**：落在目标行中央，成为目标任务的子节点。
- **DROP-003**：向左移动至少 20px 且没有更明确的同级落位时，退出当前父节点一层。
- **DROP-004**：会形成循环、超过三级或没有产生变化的落位不可提交。
- **DROP-005**：一次拖动只产生一次 store 命令、一次撤销快照和一次延迟保存。
