# 数据恢复与集成行为书

## 持久化与恢复

- **DATA-001**：页面写入和工作区元数据写入都使用乐观版本；冲突时服务端数据保持权威，客户端刷新最新状态并要求用户重试。
- **DATA-002**：恢复回收站页面必须校验工作区 revision；冲突后刷新工作区，不得按普通 HTTP 错误继续使用陈旧 revision。
- **DATA-003**：合并页面是一个仓储级事务：源页面全部任务写入目标页与源页面从元数据删除必须同时提交或同时回滚。提交前保留两页备份、源页 tombstone 和恢复 journal。
- **DATA-004**：所有恢复点按数量和字节双重限额保留，并至少保留最新一份。

## 推荐与 MCP

- **MCP-001**：界面与 MCP 使用同一默认推荐语义：只考虑依赖已完成的任务，`doing` 优先，其次按可解锁的下游任务数降序。
- **MCP-002**：MCP 写入必须通过服务端领域校验，不得绕过层级、依赖 DAG、容量或父任务完成规则。
